# 02. Growth Strategy — グロース・リテンション戦略

作成日: 2026-07-07

---

## 1. グロースモデル(AARRR)

```
Acquisition(獲得) → Activation(活性化) → Retention(継続) → Referral(紹介) → Revenue(収益)
     ↑___________________________________________________________________________|
                          グロースループ(紹介が獲得に戻る)
```

本サービスの理想ループ:
**SNS/SEOで無料診断へ流入 → 「当たってる」体験で活性化 → 毎日の運勢で継続 → 結果をSNSシェア/友人に紹介 → サブスク課金**

## 2. Activation(初回体験)設計 — FTUE(First Time User Experience)

調査より、**初回起動での体験が良いアプリは、そうでないアプリと比べ300%以上リテンション率が高い**。本サービスのFTUE(自分のこと/恋愛/仕事の初回診断)は既にモザイク+CTAの型が実装済みだが、グロース観点で以下を強化する:

- 初回診断は**60秒以内**に「当たってる」と感じさせる(既に演出2秒+結果表示で達成)
- 初回結果には必ず**シェアしたくなる1行**(意外性のある一言)を含める
- 初回離脱ポイント(入力フォームでの離脱)を計測し、フォーム項目を最小化する

## 3. Retention(継続率)設計

### Day1〜Day30の起動動機マップ

| タイミング | 起動動機 | 実装状況 |
|---|---|---|
| Day0(初回) | 診断結果を見る | ✅実装済み |
| Day1 | 「今日の運勢」を見る(毎日変わるスコア) | ✅実装済み(占いエンジン稼働中) |
| Day3 | LINE通知(◯◯さんの運気が動いています 等) | 要強化(Marketing-Issue参照) |
| Day7 | 週の振り返り(「今週はどうだった」通知) | 未実装 |
| Day14 | 無料枠切れ→サブスク検討タイミング | ✅実装済み(1回/日→もっと占う導線) |
| Day30 | 月次サマリー・継続特典 | 未実装 |

**業界平均Day30継続率は約29%**。本サービスは「毎日の運勢」という日次起動動機が既にコア機能として存在するため、**参加型コンテンツによる継続率底上げ**の条件は満たしている。今後の課題は「通知でどれだけ思い出させられるか」。

### リテンション施策の優先度

1. **プッシュ通知/LINE通知の高度化**(既存の`NotificationSetting`/`NotificationLog`モデルは実装済み。配信内容の設計が肝)
2. **連続利用の可視化**(ストリーク表示: 「7日連続で運勢をチェック中」等のゲーミフィケーション)
3. **月次/週次のパーソナルサマリー**(既に「今週・今月・来月」の期間別レポートは実装済みなので、そのダイジェスト版をLINEで送る設計)

⚠️ 注意: 調査により「ログインボーナスや無料ガチャ的な機械的インセンティブは効果が薄い」ことが分かっている。**ユーザーの気持ちを無視した機械的な数字獲得はしない**方針を維持する(既存のキャラクター人格・非開示原則とも整合)。

## 4. Referral(紹介)設計

- 診断結果のSNSシェア機能は実装済み(`ShareButtons.tsx`)。ここに**個人紐付けの紹介コード**を追加することで、UGCがそのまま獲得チャネルになる
- v5仕様で「友達紹介」UIは一度廃止されたが、`referral` APIは後方互換で残置されている(前回の監査で確認済み)。**グロース戦略上、紹介制度の再設計は価値が高い**ため、Marketing-Issueで再実装を提案する(単なる復活ではなく、SNSシェア連動の新設計として)

## 5. Revenue(収益化)とグロースの接続

- 無料会員のチャット1回/日 → 2回目で「もっと占う ※初月500円」という価格アンカリングは既存実装済み。これは**業界標準のフリーミアム設計**として妥当
- グロース観点で追加すべきは「**課金の瞬間をSNSシェアの瞬間にもする**」設計(例: サブスク登録者限定コンテンツの一部を「共有すると解放」にする等、UGCと収益化を接続する施策)

## 6. 北極星指標(North Star Metric)の提案

**「週内に3回以上、今日の運勢を開いたユーザー数」**

これは単純なDAU/MAUより、「習慣化できているユーザー」を直接捉える指標。この数値の増減を、SNS施策・通知施策・UI改善のすべての評価基準にする。
