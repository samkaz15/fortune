## 6. リスクと制約(正直な報告)

| リスク | 内容 | 対応状況 |

| **Push通知の実配信不可** | Web Push(VAPIDキー)・LINE Messaging API(チャネルアクセストークン)の認証情報が未提供のため、リマインド通知は「判定・ログ記録」までで実際の配信ができない | 認証情報提供を待つ。設計上は認証情報が来ればworker接続のみで完成する状態にしてある |

| **リマインド通知の頻度設計** | 「毎日通知」は通知疲れのリスクがあるため、既存のGM3知見(高スコア日のみ通知でCTR維持)とバランスを取る設計にした。効果は実際の配信開始後でないと検証できない | 計測基盤で効果測定できる状態にはなっている |
