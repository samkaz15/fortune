# 05. Referral & UGC Strategy — 口コミ戦略

作成日: 2026-07-07

**背景データ**: 占い利用者の31%がSNSで結果をシェアする(2026年調査)。本サービスは既に`ShareButtons.tsx`(X/Instagram/TikTok/LINE共有)を実装済みであり、**土台は既にある**。ここに「紹介の仕組み」を掛け合わせることで、口コミが自然発生する構造を作る。

---

## 1. 口コミが自然発生する仕組み(実装案一覧)

| 仕組み | 内容 | 実装状況 |
|---|---|---|
| **SNSシェア(結果画像)** | 診断結果をブランド化されたOGP画像として生成、シェア | 一部実装(ShareButtons)。OGP画像の自動生成は未実装 |
| **紹介制度** | 紹介コード経由の登録でお互いに特典(v5で一度UI廃止、APIは後方互換で残置) | 再設計が必要(Marketing-Issue参照) |
| **実績バッジ** | 「7日連続チェック」「初参拝バッジ」等のゲーミフィケーション表示 | 未実装 |
| **毎日の運勢のシェア習慣化** | 「今日の運勢」画面に常設シェアボタン+シェア専用の一言コピー自動生成 | シェア機能はあるが専用コピー生成は未実装 |
| **連続利用の可視化** | ストリーク表示でSNS投稿の材料を提供(「30日連続達成!」) | 未実装 |
| **ランキング** | 人気占いランキング(既存の`PopularRanking`コンポーネント)を「みんなが今見ている」の社会的証明として活用 | ✅実装済み、露出強化の余地あり |

## 2. 優先実装順(ROI高い順)

1. **OGP画像の自動生成**(診断結果ごとにブランド化された画像を動的生成し、SNSシェア時のサムネイルにする) — シェアされた投稿の見栄えが口コミの質を左右する最重要施策
2. **シェア専用コピーの自動生成**(結果に応じて「〇〇な一日になりそう」等の短い一言をシェアボタン押下時に自動でクリップボードにコピー)
3. **紹介制度の再設計**(v5で廃止したUIを、SNSシェア連動の新しい形で復活。「紹介した人・された人の両方に得がある」設計にする)
4. **連続利用ストリーク表示**(マイページに「◯日連続」表示。それ自体がスクリーンショットでシェアされる材料になる)
5. **実績バッジ**(初回参拝、初回サブスク、100日継続等の節目を可視化)

## 3. 口コミの質を左右する設計原則

- **ネガティブな驚き型ではなく、ポジティブな共感型**にする(占い調査の「不安煽り系キラー文言」は本サービスのブランド指針(常にポジティブ)と衝突するため不採用。既存の`conversion_spec.md`の判断を踏襲)
- シェアされる画像・テキストには**必ずサービス名とURLを含める**(UGCが自然に指名検索を生む設計)
- 紹介特典は「無料クレジット」等、サブスク導線を邪魔しない形にする(既存の「アフィリエイトはサブスク導線を妨げない」原則と同様の考え方)

## 4. 測定指標

- **バイラル係数(K-factor)**: 1人のユーザーが平均何人を紹介するか。K>1で自律的な成長ループが成立
- SNSシェア率(診断完了ユーザーのうちシェアボタンを押した割合)
- 紹介コード経由の新規登録率
