## 調査対象サービス

調査対象を選定し、各サービスの画面構成・UI・導線を分析しました。

### 1. **占いの館**（老舗：占い店舗連鎖）
- **URL**: https://uranai-no-yashiki.jp
- **提供形式**: Web + モバイルアプリ + 実店舗
- **主要機能**: 電話占い、メール占い、来店予約

#### 画面構成
```
トップ
├─ 占い師紹介（一覧・検索）
├─ 占いメニュー（タイプ別）
├─ マイページ
│  ├─ 利用履歴
│  ├─ ポイント残高
│  └─ お気に入り占い師
├─ 課金・決済
└─ 店舗情報
```

#### UI/UX特性
- **配色**: 紫・金色（神秘的）
- **フォント**: セリフ体（高級感）
- **レイアウト**: グリッド型（占い師顔写真を大きく表示）
- **CTA**: 「今すぐ占う」ボタン（赤、浮き出し効果）
- **導線**: 占い師検索 → プロフィール詳細 → 予約・決済

#### 主要課題
- 占い師検索が階層深い
- 価格体系が複雑（ポイント制）
- モバイルで文字が小さい
- ログイン後の操作がやや複雑

---

### 2. **LINE占い**（大手：ポータル型）
- **URL**: https://tarot.line.me
- **提供形式**: LINE Mini App + Web
- **主要機能**: 相性診断、毎日の運勢、占い師相談、タロット

#### 画面構成
```
タブナビゲーション (下部固定)
├─ ホーム
│  ├─ 毎日の占い（スクロール）
│  ├─ 相性診断 (Quick Link)
│  └─ おすすめ占い師
├─ 占い一覧
│  ├─ カテゴリ（運勢、恋愛、仕事）
│  ├─ 占い師
│  └─ 検索
├─ マイページ
│  ├─ プロフィール
│  ├─ 購入履歴
│  ├─ お気に入り
│  └─ ポイント
└─ その他
```

#### UI/UX特性
- **配色**: 白ベース + グリーンアクセント（親しみやすい）
- **フォント**: サンセリフ（モダン）
- **レイアウト**: カード型（スマホファースト）
- **CTA**: 「占う」ボタン（緑、サイズ大）
- **導線**: 短い・階層少ない（3タップで占い開始）
- **通知**: プッシュ通知で再訪促進

#### 主要強み
- **スマホ最適化**: タップターゲット大、スクロール重視
- **手軽さ**: ログインなしで試し占い可能
- **社会的証明**: 結果画面で「◎◎さんも診断しました」
- **リテンション**: 毎日異なるコンテンツ

#### 主要課題
- 占い師の信頼性情報が少ない
- 料金が明確でない（クレジット制）
- 都度購入がストレス

### 3. **Yahoo!占い**（大手ポータル）
- **URL**: https://fortune.yahoo.co.jp
- **提供形式**: Web + Yahoo! Japan アプリ連携
- **主要機能**: 毎日の運勢、相性診断、占い師コラム

#### 画面構成
```
ヘッダー + メインコンテンツ
├─ 本日の運勢（12星座一覧 + スクロール）
├─ 相性診断
├─ 運勢ランキング
├─ 占い師ブログ
├─ コラム
└─ 占い館（有料占い師）
```

#### UI/UX特性
- **配色**: Yahoo!ブランド色（黄+紫）
- **レイアウト**: Web標準（PC優先）
- **導線**: 検索中心（Google的）
- **信頼性**: Yahoo! Japan ブランド活用

#### 主要課題
- モバイル対応がやや弱い
- 毎日訪問のきっかけが弱い（ランキング程度）
- 占いの多様性が限定的

---

### 4. **ウラナ**（新興：AI占い特化）
- **URL**: https://urana.jp
- **提供形式**: Web + モバイルアプリ
- **主要機能**: AI占い、チャット相談、運勢予測

#### 画面構成
```
ホーム
├─ 今日の運勢（AI生成）
├─ 相談チャット（AI）
├─ 月間運勢予測
├─ プロフィール設定
└─ マイページ
```

#### UI/UX特性
- **配色**: グラデーション（モダン）
- **フォント**: フレンドリー
- **レイアウト**: チャット重視
- **CTA**: 「相談する」（会話型）
- **導線**: 即座にチャット開始可能

#### 主要強み
- **AI活用**: 24時間対応
- **レスポンス**: リアルタイム返信
- **ユーザー層**: 若年層（20-30代）
- **コスト感**: 低価格・クレジット小額

---

### 5. **ココナラ占い**（C2C型）
- **URL**: https://coconala.com
- **提供形式**: Web + モバイルアプリ
- **主要機能**: 個人占い師のマッチング、相談サービス

#### 画面構成
```
ホーム + マーケットプレイス
├─ 占い師検索（スキル検索）
├─ メッセージチャット
├─ 契約・決済
└─ レビュー・評価
```

#### UI/UX特性
- **配色**: オレンジ（活動的）
- **レイアウト**: マーケットプレイス型
- **導線**: 検索 → プロフィール → チャット → 決済
- **信頼**: ユーザーレビュー・評価

#### 主要強み
- **多様性**: 数百人の占い師が登録
- **透明性**: 価格・実績が明確
- **エスクロー**: 決済の安全性

## 主要な共通パターン

### 画面構成の共通要素

1. **ホーム画面（エントリーポイント）**
   - 毎日の運勢を上部に配置
   - 相性診断などの無料サービスをプロモート
   - パーソナライズされたコンテンツ表示

2. **占い師/サービス検索**
   - カテゴリ別分類（恋愛、仕事、人間関係等）
   - 検索バー（キーワード検索）
   - フィルタリング（価格、評価等）

3. **詳細画面（占い結果/占い師プロフ）**
   - 画像（占い師顔写真またはイラスト）
   - テキスト説明（占い内容、占い師経歴）
   - CTA（「占う」「相談する」「予約」）

4. **マイページ/プロフィール**
   - 利用履歴、購入ポイント、お気に入り
   - 設定情報（生年月日、血液型等）

5. **決済画面**
   - 金額表示、支払い方法選択
   - 完了画面での結果表示

### UI/UX共通パターン

| パターン | 採用サービス | 効果 |
|---|---|---|
| カード型レイアウト | LINE占い、ウラナ | スマホ対応、スクロール重視 |
| タブナビゲーション（下部固定） | LINE占い、ココナラ | 導線シンプル化、アクセス性向上 |
| グリッド型（占い師列表） | 占いの館、ココナラ | スキャンしやすい、比較容易 |
| チャット型インターフェース | ウラナ | エンゲージメント向上、継続利用 |
| グラデーション背景 | ウラナ | モダン感、若年層訴求 |

## 主要な課題/ギャップ

| 課題 | 現状 | 改善機会 |
|---|---|---|
| **初回ユーザー体験** | 多くのサービスで登録・課金が必須 | ログインなし試し占いで初期障壁低減 |
| **毎日訪問動機** | 「毎日の運勢」のみが多い | 通知、ゲーム化、ランキングで継続性向上 |
| **占い精度への信頼** | 占い師の実績・評価表示が少ない | 生年月日分析、結果の詳細化、根拠示唆 |
| **チャット相談の普及** | ウラナ以外は少ない | AI + 人間占い師のハイブリッド |
| **課金導線の透明性** | クレジット制で料金が曖昧 | 明確な固定価格 + サブスク |
| **神社/ジャンル連携** | なし | 神社参詣推薦、地域ベース コンテンツ |
| **ソーシャル機能** | ランキングのみ | シェア・紹介制度、コミュニティ |

## 調査結果の要点

### 国内占い市場の特性

1. **成熟市場**: 5つ以上の大手サービスが存在
2. **多様な利用形式**: Web、アプリ、チャット、占い師検索
3. **課金モデルの二極分化**: 低価格クレジット制 vs 高価格ポイント制
4. **若年化傾向**: LINE占い、ウラナなど新興サービスが若年層を獲得

### UI/UX進化の方向性

1. **スマホファースト**: カード型、タブナビゲーション
2. **チャット化**: テキスト中心の会話インターフェース
3. **パーソナライズ**: 生年月日、過去の占い履歴を活用
4. **ゲーム化**: ランキング、毎日チェック機構

### 参入の際の差別化ポイント

1. ✅ **チャットAI + 人間占い師のハイブリッド** → ウラナ以外いない
2. ✅ **神社連携・地域コンテンツ** → 完全未開拓
3. ✅ **明確な課金・サブスク** → LINE占いはできていない
4. ✅ **毎日訪問機構の強化** → 運気カレンダー、ランキング、通知
5. ✅ **占術の多様化** → 四柱推命 + 算命学 + 姓名判断の統合
