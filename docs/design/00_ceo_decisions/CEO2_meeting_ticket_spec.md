## 決定事項

```
実施形式: 公式LINEの電話（LINE公式アカウント経由の音声通話）
所要時間: 1時間
対応者: CEO本人
```

## 予約フロー

```
1. 開催日は固定（決め打ち）。ユーザーは「この日の占いに参加する」枠に対して入札する
2. 入札は「現在の最高落札金額 + 100円以上」からのみ受け付ける
   （旧仕様：現在価格を1円でも上回れば入札可 → 変更：最低100円刻みの増分を必須にする）
3. 「決済する」ボタンで入札を確定する(入札＝即決済ではなく、入札額の保留 → 落札確定後に本決済、
   という流れは既存のBid/AuctionTicket設計を維持)
4. 落札確定後、公式LINEアカウントを通じて日程・通話方法の案内を送る(Phase1では手動運用、
   Phase2で自動化を検討)
```

## 実装への影響

- `src/app/api/billing/auction/bid/route.ts`：入札額のバリデーションを
  `amount > currentPrice`(1円単位)から `amount >= currentPrice + 100`(100円単位)に変更する
- `src/app/auction/page.tsx`：入力欄のプレースホルダー・バリデーションメッセージを100円刻みに更新
- `docs/design/02_brand_strategy/GPT5_billing_copy.md`：面談チケットの説明文に
  「公式LINE電話・1時間」を明記する形へ更新
- LINE公式アカウント連携そのもの(自動メッセージ送信等)はPhase2のスコープ
  (WBS: 拡張機能「LINE連携」)。Phase1では落札後の案内は手動運用とする
