# CEO決定: 占い解釈辞書設計（Core Mapping Spec）

**承認者**: CEO / **承認日**: 2026-07-05 / **ステータス**: ✅ 確定

## 前提定義（最重要）

占いは「未来予測」ではなく **「ユーザーの状態を構造化し、次の行動を整理するための入力データ」**。
本質は占いではなく **「意思決定のUI設計」**。

## Q1: 流派の扱い

流派は統合せず **「計算層」と「解釈層」を分離** する。
- 計算層: 完全固定（暦・天体・干支）= `shichu.ts` / `rekichuu.ts` / `horoscope.ts` / `seimei.ts` / `sanmei.ts`
- 解釈層: プロダクト独自ルールの辞書 = `interpretation-dictionary.ts`
- 流派は一切UI/UXに出さない

## Q2: 「正しさ」の担保

「正しさ」は存在しない前提で設計する。代わりに:
1. **一貫性**（同じ入力＝同じ出力）
2. **再現性**（毎回ブレない）
3. **UX整合性**（行動につながる）

## Q3: 「戊午は転職向き」問題

- NG: 向いている/向いていない の断定
- OK: 状態の定義 → 行動の提案
- 例: 戊＝安定・基盤・維持力 →「今は変化より整理が向く」→「転職なら準備フェーズ」

## Q4: 占い結果の構造（絶対固定フォーマット）

1. 状態（現在）
2. 傾向（性質）
3. 注意点（**1つ**）
4. 行動（**1つだけ**）

→ 結論は必ず1アクションに収束。

## Q5: 恋愛と仕事の違い

- 恋愛＝関係の整理（感情UX）
- 仕事＝意思決定整理（認知UX）

## Q6: 複数占術の統合

重み付けスコア化して統合する（**ユーザーには非公開**）:
- 四柱推命: 0.5 / 五行: 0.3 / 天体: 0.2

## 禁止ルール

- 転職向き/不向きの断定禁止
- 未来確定表現禁止
- 流派説明禁止
- スピリチュアル表現禁止

## 実装マッピング

- 辞書本体: `src/lib/fortune-engine/interpretation-dictionary.ts`
- プロンプト統合: `prompts/chat/system_prompt.v2.4.md`（4部構成フォーマット+禁止ルール）
- 既存LANGUAGE_DESIGN（決め打ちスタイル）との整合: **状態・傾向は言い切ってよい**。
  禁止されるのは「向き/不向きの断定」と「未来の確定」のみ（Q3の区別）
