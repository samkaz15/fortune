# CEO決定: 無料枠「1日5回」のカウント定義（確定版）

**承認者**: CEO
**承認日**: 2026-07-05
**ステータス**: ✅ 確定

---

## 定義

```
1日5回 = 相手(糸町の少年)からのチャット返信が届いた回数
```

ユーザーが送信した回数ではなく、**糸町の少年からの返信をユーザーが受け取った回数**でカウントする。

## 実装への落とし込み

| ケース | カウント |
|---|---|
| 質問を送り、占い返信が正常に届いた | ✅ 1回消費 |
| 質問を送ったが、生成エラーで返信が届かなかった | ❌ 消費しない（**消費済み分を払い戻す**） |
| 上限到達後の402エラー応答 | ❌ 消費しない（返信ではなくエラー） |
| crisis検知の固定文面（相談窓口案内） | ❌ 消費しない（安全対応は課金・回数の対象外。定義の例外として明記） |

## 技術実装

- 消費はこれまで通り生成前にアトミックに行う（同時リクエストでの上限突破を防ぐため）
- **生成が失敗した場合は、消費したリソース（無料枠 / ポイント / クレジット）を払い戻す**
  - 無料枠: `refundDailyFreeQuota()`（Redisカウンターをデクリメント）
  - ポイント/クレジット: +1で戻し、取引履歴に `refund` レコードを残す
- 対象コード: `src/app/api/chat/route.ts`, `src/lib/redis.ts`
