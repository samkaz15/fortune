# 電話占いオークション機能「トークション」（MVP）仕様書

**受領日**: 2026-07-05（CEO提供） / **ステータス**: ✅ 実装済み（本ドキュメントが正）

## 目的

AI占いサービス内に「電話占いオークション機能」を実装する。
MVPでの目的は、開発コストと保守コストを最小限に抑えながら、ユーザーがオークション形式で電話占いを落札できる仕組みを構築することである。

## 基本コンセプト

- 占い師が定期的に電話占いを出品する
- ユーザーは24時間の間に入札できる
- 最も高い金額を入札したユーザーが落札する
- 落札後に決済を行う
- Googleカレンダーの空き枠から希望日時を選択する
- 当日は占い師が公式LINEから手動で電話をかける
- 電話機能・チャット機能は実装しない

## 出品ルール

出品は週2回のみとする。

- 毎週 月曜日 7:00 開始（月曜7:00〜火曜7:00）
- 毎週 金曜日 20:00 開始（金曜20:00〜土曜20:00）

開催時間は24時間固定。終了時間になったら自動でオークションを終了する。

実装しない: オークション延長 / 自動キャンセル / 自動再出品

## オークションページ構成

- **セクション1**: 占い師プロフィール / 現在価格 / 残り時間 / あなたの入札価格 / 「落札する」ボタン
- **セクション2**: 相談できる内容（例: 恋愛・復縁・結婚・仕事・転職・人間関係・金運・経営・その他）。管理画面から追加・編集可能な設計とする
- **セクション3**: レビュー（星評価・良い評価・悪い評価）。悪い評価も削除せず表示し、透明性を重視する
- **セクション4**: 免責事項（占いは未来を保証しない / 医療・法律等の専門的判断は行わない / キャンセル・返金不可 / 内容をご理解のうえ利用）

## 入札フロー

「落札する」ボタン押下後、確認モーダルを表示する。

- 表示: 現在価格 / 入札金額入力欄
- チェックボックス: □ キャンセル不可であることを理解しました / □ 免責事項に同意します
- 「確定する」ボタン押下後に入札を確定する

## 入札ルール

- 現在価格以上のみ入札可能
- 自分の入札価格を表示
- 自分が最高入札者の場合は「現在最高入札中」と表示
- 他ユーザーに抜かれた場合は画面を更新する

## リアルタイム更新

必要最小限とする。更新対象: 現在価格 / 残り時間 / 自分の入札価格 / 自分が最高入札者かどうか。
入札履歴や閲覧人数などは実装しない。ページリロード不要でリアルタイム反映される設計とする。

## オークション終了処理

24時間経過時に自動実行する。

1. オークション終了 → 2. 新規入札停止 → 3. 最終価格確定 → 4. 最高入札者取得 → 5. 落札者決定 → 6. 決済待ちステータスへ変更

同額入札の場合は、先に入札したユーザーを優先する。

## 決済

- Stripe（クレジットカード）
- 銀行振込

決済完了後のみ予約画面へ遷移する。

## 日程予約

決済完了後にGoogleカレンダーと連携する。Googleカレンダーの空き時間のみ取得し、ユーザーへ表示する。ユーザーは表示された候補から希望日時を選択する。予約完了後は日時を確定する。

## 当日の流れ

予約時間になったら占い師が公式LINEから手動で電話をかける。システム側で電話機能は実装しない。LINE通話API等の実装も不要。

## 実装しない機能（MVP対象外）

チャット機能 / アプリ内通話 / LINE API連携 / オークション延長 / 自動キャンセル / 自動再出品 / 入札履歴表示 / 閲覧人数表示 / 自動リマインド / 自動電話発信

## セキュリティ要件（最重要）

1. **サーバー側バリデーション**: フロントの価格を信用しない（開催中か / 現在価格以上か / 認証済みか / 不正リクエストでないか）
2. **サーバー時刻管理**: 終了判定はサーバー時刻を使用
3. **同時入札対策**: トランザクション/排他制御で最高額を安全に更新。同額は先着順
4. **API保護**: 開催時間外・不正価格・未認証は必ず拒否
5. **通信**: すべてHTTPS
6. **監査ログ**: 入札・落札・決済・予約の履歴を保存し管理者が確認できるようにする

## パフォーマンス

リアルタイム更新は必要最小限のデータのみ送信（現在価格・残り時間・自分の状態に限定）。

## 実装マッピング（Claude追記）

| モジュール | 実装先 |
|---|---|
| 1. オークション画面(UI) | `src/app/auction/page.tsx`（4セクション+確認モーダル+5秒ポーリング） |
| 2. 入札API | `src/app/api/billing/auction/bid/route.ts`（同意2チェック必須・現在価格以上・同額先着・楽観ロック・監査ログ） |
| 3. リアルタイム更新 | `GET /api/auction/status`（軽量ポーリング・serverNow基準の残り時間） |
| 4. 終了処理 | `src/lib/talkauction.ts` closeExpiredAuctions + `POST /api/auction/close`（CRON_SECRET）+ status API側のlazy実行 |
| 5. 決済 | `POST /api/auction/pay`（Stripe Checkout / 銀行振込）+ webhook `kind:"auction_win"` で即時paid化 |
| 6. カレンダー連携 | `src/lib/calendar-adapter.ts`（freeBusy API。未設定時はモック空き枠にフォールバック） |
| 7. 予約画面 | `src/app/auction/reserve/page.tsx` + `GET /api/auction/slots` + `POST /api/auction/reserve` |
| レビュー | `GET/POST /api/auction/reviews`（落札者のみ投稿可・削除しない） |
| 管理（相談内容/プロフィール編集） | `PATCH /api/admin/auction`（x-admin-secret） |
| スケジュール | `src/lib/talkauction.ts` nextScheduleWindows（月7:00/金20:00 JST・24h） |

### 運用メモ
- 銀行振込は「振込案内→管理者が入金確認→statusをpaidへ手動更新」の運用（MVP範囲。専用管理APIは未実装、必要になれば PATCH /api/admin/auction を拡張）
- Googleカレンダーは `GOOGLE_CALENDAR_ID` + `GOOGLE_API_KEY` 設定で実連携、未設定時は翌7日間の10:00/14:00/20:00(JST)をモック枠として表示
