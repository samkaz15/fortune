## 概要

### AI Development OS のミッション

```
複数の AI が GitHub を中心に連携し、
プロダクト開発全体（企画 → 設計 → 実装 → テスト → リリース）を
自動化・加速化する開発基盤を構築する
```

### AI ポートフォリオ

| AI | 役割 | 特性 | 主要な責務 |
|---|---|---|---|
| **Claude** | **CTO・アーキテクト** | 長期文脈・システム思考・コード実装 | 設計・実装・全体統制 |
| **ChatGPT-4** | **PM・ライター** | テキスト生成・ビジネス思考 | 要件定義・文案・プロンプト |
| **Gemini** | **マーケター・アナリスト** | マルチモーダル・検索・分析 | 市場調査・競合分析 |
| **Sakana AI** | **データサイエンティスト** | 機械学習・最適化 | 推薦・スコアリング・分析 |

## Claude の役割

### 基本スタンス

**Claude は、このプロジェクトの「CTO・主実装者」です。**

全体のアーキテクチャ・設計決定・実装品質を最終責任として担う。

### 主な責務

#### 1. 設計フェーズ
- システムアーキテクチャの策定
- 技術スタック選定・検証
- API 設計
- DB スキーマ設計
- インフラ構成設計

#### 2. 実装フェーズ
- バックエンド実装（FastAPI + Python）
- フロントエンド実装（Next.js + React/TS）
- テスト実装（Unit/Integration/E2E）
- インフラ実装（Terraform）

#### 3. 品質保証・リファクタリング
- コードレビュー（自身と他の AI の出力）
- テスト全般の品質監視
- パフォーマンス最適化
- 技術債管理

#### 4. 全体統制
- Phase マイルストーン管理
- 他の AI との連携調整
- CEO / ステークホルダーとの技術相談
- Risk 管理・意思決定

### 作業サイクル

```
┌─────────────────────────────────────┐
│ 1. Issue を受け取る                  │
│ (WBS または ChatGPT からの引き継ぎ)  │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 2. 設計・仕様を確認                   │
│ (API/DB 設計書を確認)                │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 3. 実装                              │
│ (コード生成、テスト付き)              │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 4. セルフレビュー                     │
│ (コード品質、テストカバレッジ確認)    │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 5. PR 作成                           │
│ (ドキュメント + コード提出)           │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 6. 人間によるレビュー                 │
│ + 自動テスト実行                      │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 7. Merge & Deploy                   │
└─────────────────────────────────────┘
```

### 入力・出力仕様

#### 入力
- **GitHub Issue** (WBS、Feature Issue)
- **API/DB 設計書** (ChatGPT ドラフト → Claude 確定)
- **テスト要件** (ChatGPT が提案)
- **フィードバック** (PR レビューコメント)

#### 出力
- **実装コード** (Backend/Frontend/Infrastructure)
- **テストコード** (Unit/Integration/E2E)
- **技術ドキュメント** (API 詳細仕様、実装ガイド)
- **アーキテクチャ更新** (新技術採用時)

### 依存・連携

```
Claude ←→ ChatGPT: 要件・設計の確認
Claude ←→ Gemini: 競合分析・市場情報の参照
Claude ←→ Sakana AI: 推薦エンジンの実装指示
Claude → GitHub: 成果物の commit・PR 作成
```

## ChatGPT-4 の役割

### 基本スタンス

**ChatGPT-4 は、このプロジェクトの「PM・ライター・要件家」です。**

ビジネス要件をテキスト形式で整理し、Claude が実装しやすい形に変換する。

### 主な責務

#### 1. 要件定義フェーズ
- ビジネス要件の整理・テキスト化
- PRD（Product Requirements Document）作成
- ユーザーストーリー・ユースケース記述
- 受理基準（Acceptance Criteria）定義

#### 2. 設計支援
- UI/UX コンセプト・文言設計
- プロンプト設計（AI チャット用）
- テストシナリオ・テストケース作成
- ドキュメント初期ドラフト

#### 3. マーケティング・通知文言
- 課金画面のコピーライティング
- 通知文言・メール文言生成
- ニュースレター・公式メッセージ作成
- SNS 投稿企画

#### 4. テスト・QA 支援
- テストデータの生成
- テストシナリオの多様化
- エッジケースの提案
- UI テストの自然言語記述

### 作業サイクル

```
┌─────────────────────────────────────┐
│ 1. Issue / 競合分析を受け取る         │
│ (Gemini からの情報)                  │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 2. ビジネス要件を整理                 │
│ (PRD ドラフト作成)                   │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 3. Claude に要件提出                 │
│ (PR for Design)                     │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 4. Claude のフィードバックを反映     │
│ (技術的実現可能性の確認)              │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 5. 承認申請                          │
│ (CEO/PM からの承認)                  │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 6. 実装前の補助                      │
│ (プロンプト設計、テストケース)        │
└────────────┬────────────────────────┘
```

### 入力・出力仕様

#### 入力
- **Gemini の分析レポート** (市場・競合情報)
- **CEO のビジョン・方針**
- **Claude の技術制約**
- **ユーザーフィードバック**（Phase2 以降）

#### 出力
- **PRD（Product Requirements Document）**
- **UI/UX 文言・コピー**
- **プロンプト設計書** (AI チャット用)
- **テストシナリオ・テストケース**
- **UI テストの自然言語記述**

### 依存・連携

```
ChatGPT ←→ Gemini: 市場情報の活用
ChatGPT ←→ Claude: 技術実現可能性の確認
ChatGPT → GitHub: PRD・要件の提出
```

## Gemini の役割

### 基本スタンス

**Gemini は、このプロジェクトの「市場調査官・データアナリスト」です。**

市場・競合・トレンド・ユーザー行動を分析し、戦略的な決定根拠を提供する。

### 主な責務

#### 1. 事前調査フェーズ（Phase 0）
- 国内占いサービスの UI/UX 分析
- 海外 AI 占い・診断サービスの分析
- 課金モデル・LTV 構造の分析
- ユーザー行動パターンの分析

#### 2. 戦略提案
- 差別化ポイントの抽出
- 競合強み・弱み分析
- 市場トレンド分析
- マーケティング・グロース戦略案

#### 3. 継続的なマーケット監視
- β版・リリース後の競合分析
- ユーザー行動データの分析
- 改善提案の根拠データ提供

#### 4. コンテンツ調査
- SEO キーワード調査
- SNS バズパターン調査
- メディア・PR トレンド分析

### 作業サイクル

```
┌─────────────────────────────────────┐
│ 1. Issue を受け取る                  │
│ (調査タスク)                         │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 2. Web 調査・スクレイピング           │
│ (競合サイト、App Store等)            │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 3. データ整理・分析                   │
│ (スプレッドシート、チャート)         │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 4. 分析レポート作成                   │
│ (ChatGPT が要件化)                   │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 5. GitHub に提出                     │
│ (Report + 参考 URL)                  │
└────────────┬────────────────────────┘
```

### 入力・出力仕様

#### 入力
- **調査対象** (競合サービス URL、市場キーワード)
- **分析軸** (UI/UX、課金モデル、ユーザー層)
- **データソース** (App Store、Web サイト、ソーシャルメディア)

#### 出力
- **競合分析レポート** (Markdown / スプレッドシート)
- **市場トレンド分析**
- **差別化ポイント提案**
- **グロース戦略案**
- **参考 URL 集約**

### 依存・連携

```
Gemini → ChatGPT: 分析結果の要件化
Gemini → Claude: 市場情報のアーキテクチャ反映
Gemini → GitHub: レポート提出
```

## Sakana AI の役割

### 基本スタンス

**Sakana AI は、このプロジェクトの「ML エンジニア・データサイエンティスト」です。**

ユーザー行動データに基づく推薦エンジン・スコアリング・分析を担当する。

### 主な責務

#### 1. 推薦エンジン構築（Phase 2-3）
- ユーザー行動データの取得・クレンジング
- 類似ユーザー検出アルゴリズムの開発
- 占い結果の推薦ロジック開発
- 神社レコメンデーション（場所ベース）

#### 2. スコアリング・運気計算
- 運気スコアの算出ロジック開発
- 95 点以上通知のトリガー設定
- ユーザー行動パターンのスコアリング

#### 3. ユーザー分析
- セグメンテーション（行動ベース）
- コホート分析
- LTV 予測
- チャーン率予測

#### 4. A/B テスト設計・分析
- 実験設計（仮説、メトリクス、サンプルサイズ）
- 統計的有意性判定
- 施策効果測定

### 作業フロー

```
┌─────────────────────────────────────┐
│ 1. データ要件を受け取る              │
│ (Claude から)                        │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 2. モデル設計・学習                   │
│ (特徴量エンジニアリング、学習)        │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 3. API 化・統合                      │
│ (Claude がバックエンドに統合)        │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 4. パフォーマンス・精度評価           │
│ (A/B テスト、ユーザーフィードバック) │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│ 5. 改善提案                          │
│ (次期施策、アルゴリズム改善)         │
└────────────┬────────────────────────┘
```

### 入力・出力仕様

#### 入力
- **ユーザー行動データ** (Claude が DB から提供)
- **推薦要件** (PR で記述)
- **パフォーマンス目標** (LTV、継続率等)

#### 出力
- **推薦モデル** (Python コード / モデルファイル)
- **API 仕様** (OpenAPI)
- **スコアリング公式**
- **分析レポート** (効果測定、改善案)

### 依存・連携

```
Sakana AI ←→ Claude: API 統合、データ仕様確認
Sakana AI → GitHub: モデルコード、分析レポート
```

## AI 間の連携フロー

### リアルタイム連携（WBSベース）

```
GitHub WBS Issue
    ↓
    ├─ [Design] API仕様 作成
    │  ├─ Gemini: 競合分析から差別化API設計
    │  ├─ ChatGPT: 要件定義（PRD）作成
    │  └─ Claude: API詳細仕様 確定
    │
    ├─ [Feature] 機能実装
    │  ├─ ChatGPT: テストケース生成
    │  ├─ Claude: 実装 + テスト
    │  └─ Sakana: 推薦ロジック（該当時）
    │
    ├─ [Review] コードレビュー
    │  └─ Claude: セルフレビュー + 人間レビュー
    │
    └─ [Deploy] デプロイ
       └─ Claude: デプロイ + 監視

    ↓
成果物が GitHub に commit
    ↓
全チーム（AI+人間）が成果物で次フェーズへ
```

### データフロー図

```
WBS（Excel/Sheets）
  ↓ [Gemini が読み込み]
競合分析
  ↓ [ChatGPT が要件化]
PRD (GitHub)
  ↓ [Claude が設計]
API/DB 設計書 (GitHub)
  ↓ [Claude が実装]
実装コード (GitHub)
  ↓ [ChatGPT がテストケース]
テストコード (GitHub)
  ↓ [Sakana がモデル学習]
推薦エンジン (GitHub)
  ↓
完成したプロダクト
```

## エスカレーション・意思決定

### 決定フロー

```
意思決定が必要なケース
  ↓
Technical Decision（技術判断）
  ├─ 判定者: Claude
  ├─ 相談相手: ChatGPT（ビジネス影響度）
  └─ 決定方法: Architecture ADR（Architecture Decision Record）

Business Decision（ビジネス判断）
  ├─ 判定者: ChatGPT
  ├─ 相談相手: Claude（技術実現性）、Gemini（市場情報）
  └─ 決定方法: GitHub Issue + CEO 承認

Conflict（AI間の意見相違）
  ├─ 例1: Claude「スケールのため水平分割」 vs Gemini「競合未対応」
  ├─ 決定者: Claude（CTO権限）
  └─ 最終判定: CEO
```

### Issue テンプレート（決定記録）

```markdown
## ADR: [Architecture Decision Record]

### Status
[Proposed / Accepted / Deprecated]

### Context
[背景・なぜこの決定が必要か]

### Decision
[決定内容]

### Rationale
[理由・分析データ]

### Consequences
[影響範囲・リスク]

### Stakeholders
- Claude (Approve/Reject)
- ChatGPT (Feedback)
- Gemini (Feedback)
- CEO (Final Approval)
```

## AI Cost Management

### トークン使用量管理

#### 月間予算配分（見積）

| AI | 役割 | 月間予算（目安） | 制約 |
|---|---|---|---|
| Claude | 実装・設計 | $2,000 | 長文・継続タスク |
| ChatGPT-4 | 要件・文案 | $1,000 | テキスト生成 |
| Gemini | 調査・分析 | $500 | Web 検索・分析 |
| Sakana AI | ML モデル | $300 | API 呼び出し |

#### コスト最適化戦略

```
1. バッチ処理化
   - 小タスク統合 → 1回の API 呼び出しで複数処理

2. キャッシング
   - よく参照するドキュメント（設計書など）をメモリ保持
   
3. 段階的リクエスト
   - 概要検索 → Gemini（低コスト）
   - 詳細設計 → Claude（高コスト）

4. 機械学習モデル活用
   - 反復的なタスク → ファインチューニング
   
5. 期間限定タスク
   - Phase 終了後の AI 利用を制限
```

### 使用量監視

```bash
# 月次レポート（Claude Code で自動生成）
# - 各 AI の月間トークン使用量
# - 予算と実績の比較
# - 将来予測
# - 最適化提案
```

## AI Development OS の進化パス

### Phase 1（MVP）
```
Claude（主軸） + ChatGPT（支援） + Gemini（前置き調査）
= 基本的な AI 連携フレームワーク確立
```

### Phase 2（拡張）
```
+ Sakana AI（推薦エンジン）
= マルチエージェント統合の検証
```

### Phase 3（スケール化）
```
+ 専門 AI（Domain-Specific Agent）
= 産業別対応 AI エージェントの準備
```

### 将来（他プロダクト対応）
```
Orchestrator 層の完成度向上
↓
新規プロダクト立案時に即座に適用可能
↓
「AI 駆動型開発」の標準化
```
