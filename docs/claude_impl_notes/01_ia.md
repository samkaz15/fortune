# CL1: IA改訂版（要約）

チャット統合・課金3種・オークションを反映した最終版。詳細な調査根拠・レビューは
別途Markdown設計書（IA設計書 差別化戦略反映版）を参照。ここでは実装との対応関係のみ記す。

## サイトマップ（実装対応）

```
/                    TOP（Level1）
/consult             占いチャット（Level1〜2。category未指定ならカテゴリ選択カードを表示）
/result/:id          診断結果（Level3。未ログインでも閲覧可、全文はisUnlockedで判定）
/plans               料金プラン（Level2。サブスク/クレジットをタブ切替で1画面に統合）
/plans/complete      決済完了（Level3。トースト的な最小表示→mypageへ自動遷移）
/auction             オークション一覧+入札（Level2〜3を1画面に統合）
/mypage              マイページ（Level1）
/mypage/notifications  通知設定（Level4）
/mypage/account        会員情報・支払い管理（Level4）
/mypage/account/cancel 退会手続き（Level4。休会提案のワンクッションあり）
/news                お知らせ一覧（Level1。現状ダミーデータ、Newsモデル未実装）
/auth/login /auth/signup  認証（Level4）
/legal/terms /legal/privacy /legal/tokushoho  法定表記（Level4）
```

## 設計判断の要点

- 診断メニューを5枚に分けず `/consult` 1本に統合（占術は裏側ロジックに隠す）。
- 決済導線は `/plans` にサブスク/クレジットを集約し、オークションのみ性質が異なるため
  `/auction` を独立させた（入札・カウントダウン等、他の課金導線とUIが根本的に違うため）。
- 決済完了画面を独立ページとして「見せる」のではなく、`/plans/complete` は
  1.8秒でマイページへ自動遷移する最小表示にとどめている（UX改善提案の反映）。
