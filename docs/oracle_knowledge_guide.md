# 師匠知見ベース(Oracle KB) 更新ガイド

作成日: 2026-07-12 / 対象: CEO(寒川さん)

## これは何か

「錦糸町の少年」側の記憶です。ユーザー側の記憶(人生カルテ)と対になる仕組みで、
**寒川さんが実鑑定で掴んだ独自ノウハウ**(例: 財布ブランド×運気)を構造化して蓄積し、
チャットで関連する話題が出た瞬間に「僕の見解ですが…」として展開されます。
**知見を追加すればするほど、AIの回答がニッチで真似できないものになっていきます。**

## データの場所と形式

`knowledge/oracle/` 配下のJSONファイル。1テーマ=1ファイル(現在: `wallets.json`=財布×運気)。

```json
{
  "topic": "テーマ名",
  "updatedAt": "2026-07-12",
  "source": "CEO直伝(実鑑定ノウハウ)",
  "commonPrinciple": "テーマ全体に共通する原則(任意)",
  "expressionGuideline": "AIが語るときの注意(任意)",
  "entries": [
    { "id": "一意なID", "brand": "対象名(任意)", "tags": ["検索用キーワード"], "insight": "見解の本文" }
  ]
}
```

**tagsが検索の鍵です。** ユーザーがチャットで使いそうな言葉(「ヴィトン」「金運」「財布」)を
入れてください。ここに入っていない言葉では想起されません。

## 更新方法(3通り・簡単な順)

### 方法A: このチャットで私(開発AI)に伝える【推奨・一番楽】
今回のように「ロエベは人脈」と自然文で送ってもらえれば、私が構造化してJSONに追加し、
テストを通してpushまでやります。**新テーマ(例: 「腕時計×運気」「オフィスの方角」)の追加も同様です。**

### 方法B: GitHub上で直接編集【開発者不要・5分】
1. github.com/samkaz15/fortune → `knowledge/oracle/wallets.json` を開く
2. 鉛筆アイコン(Edit)→ entriesに項目を追加 → Commit changes
3. Vercelが自動デプロイ → **数分後には本番のチャットが新しい知見で答え始めます**
※JSONの形式ミス(カンマ忘れ等)があるとそのファイルだけ無視されます(サイトは落ちない設計)。
　不安なら方法Aで。

### 方法C: 管理画面から追加【Phase 3・未実装】
知見が数十テーマを超えたら、DB化+管理画面を作ります(現状は不要)。

## 運用ルール

1. **断定してよいのは寒川さんの見解として。** ブランドの品質評価と誤読されない表現は
   `expressionGuideline` でAIに指示済み(他ブランドを貶める比較はしない・ユーザーの持ち物は肯定から入る)
2. **効果が変わった/深まったら上書きでOK。** gitが履歴を全部持っているので、過去の見解にはいつでも戻れます
3. 医療・法律・投資の具体的判断に踏み込む知見はLayer0ガードレールが優先されます(AIが自動で丸めます)

## 動作確認

チャットで「ヴィトンの財布ってどうですか?」と聞く → 少年が「単純に金運が強いです。
特徴は今の稼ぎ方の延長線上でアップが起こることで…」と**寒川さんの見立てそのもの**で答えれば成功。
