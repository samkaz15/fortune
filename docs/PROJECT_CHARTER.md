# Project Charter: AI Development OS

## Executive Summary

このプロジェクトの表面的な目的は「AI占いサービス『fortune』の開発」です。

しかし、**真の目的は「AIが主体的にプロダクト開発全体を推進できる『AI Development OS』を構築・検証すること」**です。

fortuneはこのOSを検証するための最初のPoC（概念実証）であり、将来的には他ジャンルのWebサービスやSaaSにも応用可能な再利用可能な開発基盤を目指します。

---

## プロジェクトの目的

### 短期目標（Phase 1-3: 2026/7～2027/4）
- AI占いサービス「fortune」を開発し、日本市場で事業化する
- MVPの実装（Phase1: 2026年9月）
- 正式版リリース（Phase2: 2026年11月）
- スケール対応（Phase3: 2026年12月）

### 長期目標（2027年以降）
- **AI Development OS の確立**
  - AIが要件定義から設計・実装・テスト・リリースまで主体的に推進可能なフレームワーク
  - GitHub/Git を中心とした成果物管理
  -複数AI（Claude/ChatGPT/Gemini/専門AI）の統合オーケストレーション
  - 他プロダクト領域への即時応用可能な再利用設計

---

## 背景

### 現状認識
- 日本の占い市場は確立されており、デジタル化・AI化の機会がある
- 現在の開発では、AIが特定の実装タスクのみを担当している
- AIの能力向上により、「トップレベルの企画・設計から実行まで」をAIが担当できる時代が来ている

### このプロジェクトの位置づけ
- 新しい開発パラダイム「AI駆動型開発」の実装と検証
- 人間とAIの協働モデルの確立
- スケーラブルで再利用可能な開発基盤の構築

## スコープ

### 含まれるもの

#### コア機能
- 複合占術エンジン（四柱推命・算命学・姓名判断の統合）
- AIチャット相談機能（人生相談・カウンセリング）
- ユーザー認証・プロフィール管理
- 複数課金モデル（サブスク・クレジット・オークション）
- 通知システム（運気通知・95点以上通知）

#### 拡張機能
- ビジネス占い・相性診断
- 運気カレンダー・毎月やるべきこと
- 神社レビュー・AI推薦
- 運営者コメント機能
- ユーザーランキング

#### 技術基盤
- バックエンド・フロントエンドアーキテクチャ
- データベース設計・スケール対応
- API設計（REST/GraphQL検討）
- インフラ構成（AWS/GCP）
- CI/CD パイプライン
- 監視・SRE体制

#### AI Development OS
- AI（Claude/ChatGPT/Gemini）ごとの役割定義
- GitHub運用ポリシー
- 成果物受け渡しメカニズム
- プロンプト管理・バージョン管理
- マルチエージェント統合設計

### スコープ外
- 占い理論自体の開発（CEO監修に基づくもののみ）
- マーケティング・PR活動の詳細実装
- 法務・コンプライアンス対応（方針策定のみ）
- オフライン店舗運営
- 非モバイルプラットフォーム（初期段階）
- 占いAIの学習・再学習（Sakana AIの学習は含まない）

## 開発方針

### アジャイルと計画性の両立
- **WBSベースの明確なマイルストーン**: 全タスクを可視化・GitHub Issues化
- **2週間スプリント**: 実装フェーズではスプリント単位で進捗確認
- **フェーズゲート**: 各Phase終了時に品質・機能完成度をレビュー

### AI中心のワークフロー
- **Claude Code**: 設計・実装・リファクタリングの主軸
- **OpenAI API**: 要件定義・テキスト生成・プロンプト最適化
- **Gemini API**: 市場調査・競合分析・マルチモーダル分析
- **Sakana AI**: 推薦ロジック・スコアリング（Phase2以降）
- **GitHub**: AIとメンバー間のコラボレーション・版管理

### 成果物の明確化
- 各タスクの成果物を明示（設計書・実装コード・テスト結果）
- GitHub Issues/PRで進捗・レビュー一元管理
- プロンプト・スクリプトはリポジトリ内で版管理

## 品質方針

### コード品質
- **言語別ガイドライン**: Python/TypeScriptなど言語ごとのコーディング標準
- **自動テスト**: Playwright + Jest/Pytest を基本
- **コードレビュー**: PR単位で必須
- **リファクタリング**: 継続的改善に10%のリソース配分

### 設計品質
- **アーキテクチャレビュー**: Phase開始前に設計を検証
- **技術債の管理**: スプリント単位で技術債 Issue を管理
- **ドキュメント品質**: 設計書・API仕様は実装と同期

### ユーザー品質
- **β版テスト**: 限定ユーザーによる検証（Phase2終了時）
- **A/Bテスト**: Phase3以降、UI/課金施策の効果測定
- **継続率・LTV監視**: リリース直後から監視・改善サイクル

## 技術方針

> **⚠️ 2026-07-02改訂**：本セクションは当初Python/FastAPIを想定していましたが、
> 実装は行われないまま、Claudeが並行してNext.jsフルスタックでPhase1(CL1〜CL14)を
> 実際に実装・動作確認まで完了させました。「先に動くものがある方を正とする」という
> 判断のもと、本セクションはNext.js実装に合わせて更新しています。詳細は
> [ARCHITECTURE.md](./ARCHITECTURE.md) を参照してください。

### バックエンド／フロントエンド
- **フレームワーク**: Next.js 14 (App Router) + TypeScript — フロントエンドとAPIを同一リポジトリに統合
- **DB**: PostgreSQL（Prisma ORM、Supabase推奨。将来の水平分割を想定した設計は維持）
- **キャッシュ**: Redis（Upstash推奨）
- **決済**: Stripe（サブスク/クレジット/オークション入札の3モデルに対応、Webhook実装済み）
- **UIコンポーネント**: Tailwind CSS（デザイントークンを`tailwind.config.ts`で一元管理）
- **Mobile**: 未着手。Phase1〜2はレスポンシブWebで対応し、必要になった時点でFlutter等を検討

### インフラ
- **ホスティング**: Vercel（Next.jsとの親和性、自動スケール、CDN込み）
- **IaC**: Phase1〜2では不要と判断（Terraformの導入はPhase3のインフラ複雑化時に検討）
- **CI/CD**: GitHub Actions（Lint/型チェック/テストの自動化。設定はPhase1完了後に追加予定）
- **監視**: Phase2以降でDatadog等の導入を検討

### AI運用
- **プロンプト管理**: `/prompts` で version control（GPT3設計方針を踏襲・実装済み）
- **API統合**: Sakana AI（占術結果の自然文生成）。将来的に他プロバイダを追加する場合は
  `src/lib/fortune-engine/sakana-ai-adapter.ts` のインターフェースを介して差し替える
- **コスト管理**: 月単位でトークン使用量を監視（Phase2で計測基盤を整備）
- **品質管理**: crisis検知フィルタ(`crisis-detection.ts`)等、AI出力前後のガードレールを実装済み

## 組織・役割分担

### 担当AI
| AI | 役割 | 主な責務 |
|---|---|---|
| **Claude** | CTO・アーキテクト・主実装者 | 設計・実装・リファクタ・コードレビュー |

- **プロダクトマネージャー** (必要に応じて): ビジネス要件調整
- **レビュアー**: AI生成成果物の品質確認

## リスクと対策

### 技術リスク
| リスク | 対策 |
|---|---|
| AI生成コードの品質低下 | コードレビュー・テスト自動化・段階的統合 |
| プロンプト最適化の時間超過 | プロンプト版管理・テストスイート構築 |
| スケーラビリティ問題 | Phase0での設計・早期の負荷テスト |

### スケジュールリスク
| リスク | 対策 |
|---|---|
| Phase間の手戻り | フェーズゲート・設計レビューの徹底 |
| AI APIのレート制限 | 使用量監視・バッチ処理の検討 |
| 予期しない要件変更 | CEO監修での事前確認・変更ログ管理 |

### ビジネスリスク
| リスク | 対策 |
|---|---|
| ユーザー獲得難 | β版での早期フィードバック・改善サイクル |
| 課金導線の設計ミス | A/Bテスト基盤の早期構築 |
| 占術精度への信頼低下 | CEO監修・内部テスト充実 |
